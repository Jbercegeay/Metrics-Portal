Release Policy

This changelog is updated for production-facing releases, not every commit. New production work should be recorded under Unreleased, then moved into a dated version when the release is approved and deployed.

  • Major: new department rollouts, major workflow changes, or breaking Smartsheet/config changes.
  • Minor: new features, new admin tools, new reports, or meaningful workflow improvements.
  • Patch: bug fixes, wording fixes, validation fixes, and low-risk production corrections.
