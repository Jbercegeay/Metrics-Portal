Admin Panel — Sequences
Sequences & Goals
Process types and their performance targets — drives OEE Performance metric.

📋 What is a Sequence?

  • A sequence is a type of work (e.g., Package, Spool Check, Re-Label)
  • Associates select their sequence at the start of each job
  • The selected sequence determines what goal rate is used for OEE
  • Spool Check has additional QC fields (sequence #, check #) that appear automatically

🎯 Goal (Parts/Hr) — Takt Time

  • Set the expected parts per hour for each sequence
  • This is used to calculate OEE Performance: Actual Output ÷ Goal Output
  • A higher goal = tougher Performance score
  • Set it to match your engineered cycle time or historical benchmark
Click "+ Add Sequence", fill in the name and goal, then click "Save".
Admin Panel — Dropdowns
Defects & Events

🔴 Managing Defect Types

  • Defects appear as individual buttons on the operator dashboard
  • Each tap on a defect button increments that defect count by 1
  • Defect totals feed into OEE Quality calculation
  • Add defect types that match your specific quality codes
  • Changes appear on all operator screens immediately (no reload needed)
Renaming or deleting a defect type affects all future logs. Historical data in Smartsheet is not modified.

📅 Managing Event Types

  • Events = planned and unplanned downtime categories
  • Examples: Break, Lunch, Meeting, Maintenance, Changeover
  • When an operator logs an event, that time is deducted from Availability
  • Events automatically split across PCD hour boundaries
  • The PCD tracker shows planned vs. actual minutes per event type
Keep event types clear and specific — operators select from a dropdown during active shifts.
Admin Tools
Supervisor Reset
Emergency wipe of all active portal data — used at shift change or to resolve stuck states.
How to trigger: From the Main Portal (not the Admin Panel), press:
Ctrl + Shift + X
Then enter PIN: 2026

✅ What it does

  • Wipes all active shift data from this workstation
  • Clears operator sessions, active lots, timers, PCD data
  • Reloads the portal to a clean state
  • Does not delete anything already submitted to Smartsheet

🚨 When to use

  • Shift change: If the outgoing shift forgot to End Shift Reset, use this to clear and start fresh
  • Stuck state: App is frozen or showing stale data from a previous shift
  • Testing: Need to clear test data without going through normal flow
Warning: Any in-progress, un-submitted shift data on this PC will be permanently lost. Only use it if the data is already gone or invalid.

Associate Portal
Shift Setup
Operators fill this out once at the start of each job. All fields drive the PCD tracker and Smartsheet log.
FieldDescription
Work DateDate of the production run (defaults to today)
Shift Start / EndClock times for the shift window; PCD hours are built from this range
SequenceProcess type — drives the goal rate from Admin config
Lot #Work order / lot number for this job

🔍 Spool Check — Extra Fields

  • When Spool Check is selected as the sequence, two extra fields appear automatically
  • Spool Check Seq: 10, 20, or 30 (which sequence in the spool check cycle)
  • Check #: 1st, 2nd, 3rd, 4th, or 5th inspection of this spool
  • Both fields are logged to Smartsheet alongside production data
Shift Start/End can cross midnight — the portal handles multi-day shift windows correctly.
Associate Portal
Logging Production
The primary action operators perform throughout their shift — counting good units and defects.

✅ Good Units

  • +1 button — tap once per good unit
    Keyboard shortcut: H
  • +5 button — tap for a batch of 5
    Keyboard shortcut: G
  • Running total shown as a real-time yield badge
  • Counts accumulate for the current lot until Submit Job
The keyboard shortcuts (H and G) are popular on the floor — operators can count without looking at the screen.

🔴 Defects

  • Each defect type has its own button in the defect panel
  • Tapping a defect button adds 1 to that defect count
  • Defect totals are displayed in real time
  • Defect data feeds OEE Quality: Good ÷ (Good + Bad)
  • All defects are submitted per lot to Smartsheet
Defect types are configured in the Admin Panel → Defects section. Changes appear instantly on all active dashboards.
Associate Portal
Event Entry & Timers
How operators log downtime — break, maintenance, changeovers, meetings.
1
Switch to "Event Entry" mode Toggle at the top of the dashboard switches from production counting to event logging
2
Select the event type Dropdown shows all events configured in Admin Panel
3
Start the timer — or enter time/duration manually Live timer counts up while the event is active; or enter start time and duration directly
4
Stop timer / save event Event is recorded; automatically split across PCD hour boundaries if it spans hours

🕐 How Events Affect OEE

  • Event time is deducted from available production minutes
  • This flows directly into the Availability component of OEE
  • The PCD tracker shows planned minutes vs. actual minutes per hour, with downtime broken out
Events that span an hour boundary (e.g., a 45-min break starting at 9:50) are automatically split into the correct PCD hour buckets — operators don't need to do anything special.

📤 Submit Job

  • Submits the current lot's data to Smartsheet (good count, defects, lot #, item #)
  • Resets the lot counter — ready to start a new lot
  • Use when changing to a new work order mid-shift

🔴 End Shift Reset

  • Submits the active lot + full PCD tracker + Shift End event to Smartsheet
  • Signs out all operators and clears the workstation for the next shift
  • Shows a 5-minute warning before the shift end time
  • Required at the end of every shift — do not just close the browser
