# Production-Readiness Agent Instructions

These instructions apply to all work in `docs/production-readiness/` and to implementation work performed from this playbook.

1. Read `Memory.md` before planning or changing production-readiness work.
2. Use the roadmap phase and work-package scope recorded in the playbook.
3. Update `Memory.md` whenever work changes code, documentation, requirements, decisions, migrations, configuration, tests, deployment state, risks, or next actions.
4. Update both the current-state sections and the session history before declaring work complete.
5. Include the active branch, commit or PR when available, validation performed, deployment status, blockers, and exact next action.
6. Never place secrets, credentials, employee-sensitive information, or production payloads in `Memory.md`.
7. A work package is not complete until its memory update is included in the same branch and commit.
